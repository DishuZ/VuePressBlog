# 在线视频网站

# 立项原因

# 项目角度

在本科时，曾经参与过学校慕课网站的搭建。作为一个视频网站，其作用是提供给学校现有师生一个 线上学习的平台，最重要的功能应该是围绕『视频』+『学习感』来打造。因此，作为后端开发，现在回顾这个网站项目，其实有很多可以改进的地方 （痛点）：

  • 痛点1：视频上传和播放加载缓慢 → 改进：视频上传&视频播放 ⇿ 视频分片上传（断点续传）、视频分片加载（安全性）播放、内容MD5算值（秒传）
  • 痛点2：频繁登录。 → 改进：线上学习的连续性&沉浸感 ⇿ JWT 用户认证（用户安全）、双 Token 机制（无感刷新 Token）
  • 痛点3：无学习氛围。 → 改进：学习氛围的体现 ⇿ 弹幕系统（及时提问）、关注用户消息推送（MQ发布订阅模式）
  • 痛点4：用户单一，无明显划分。 → 改进：用户&权限划分 ⇿ RBAC（Role-Based Access Control）权限控制模型

# 技术角度

以前采用的是 WAMPServer（Win+Apache+Mysql+PHP）以及WordPress，技术有点老，所以想更新后端技术栈。

并且对于一些技术点，还具备可探讨可升级的潜力，包括：视频分片上传/加载、ES搜索海量视频、用户订阅关系、视频弹幕、Redis缓存等。

# 项目架构

# 业务（功能）架构

顶层：用户服务，如 注册登陆、教师&学生权限、查找感兴趣视频等

中间层：核心功能：在线视频流播放 + 实时弹幕

底层：管理后台，如 视频上传、数据统计、系统消息推送等

顶层功能 中间层 底层功能
用户登录、用户注册 双Token机制 RCAC访问/../接口权限控制
用户关注、发布动态 发布订阅机制 RocketMQ发布订阅、流量削峰
视频上传、视频加载播放、视频秒传、视频评论 分片上传、分片加载机制 FastDFS轻量分布式存储服务
数据统计 AES、RSA、MD5加解密服务
发送弹幕、弹幕实时加载 缓存弹幕、MQ削峰机制 Redis缓存
查找感兴趣视频 推荐机制 ElasticSearch搜索

# 技术架构

选择哪些技术来实现项目的需求？

技术选型：SpringBoot2.x + Mysql + Mybatis + Maven

开发模式：采用经典MVC，控制层（Controller层）、服务层（Service层）、数据层（Dao层）

# 部署架构

项目可执行包的部署服务器设计？

前端：服务转发（对接后端服务API） + 负载均衡
后端：业务处理 + 功能实现
工具：缓存/Redis、队列/RocketMQ

# 开发环境搭建

操作系统：Windows
开发工具：IntelliJ IDEA
基础：JDK1.8、Maven

# 创建多模块、多环境项目

多模块：创建MVC模式多模块项目，控制层（controller）、服务层（service）、数据层（dao）。

  • 首先，需要创建一个 Maven 项目；File → new → project → Maven (SDK 1.8) → Next → Name(imooc-bilibili) → Finish
  • 然后，在这个 Maven 项目的下面创建三个子模块，分别代表 controller、service、dao 层。右键imooc-bilibili → New → Module → Maven （SDK 1.8) → Parent(imooc-bilibili) → Name(imooc-bilibili-api) → Finish

多环境：添加不同环境的 properties 配置文件

  • 在service的resource中创建 application.properties ，抽取项目模块中公共的环境配置
  • 在api/controller的resource中创建 application-online.properties 作为生产环境配置
  • 在api/controller的resource中创建 application-test.properties 作为测试环境配置

# 项目知识点

# JWT

组成：

  • JWT头部：声明的类型、声明的加密算法（通常使用SHA256，项目使用RSA256）
  • JWT载荷：存放有效信息，一般包含签发者、所面向的用户、接收方、过期时间、签发时间以及唯一身份标识（项目使用UserID）

# Ioc 配置的三种方式

  • xml 配置。项目中没有用到。
  • Java 配置。项目中 service/config/* 的所有类都是这种配置。
  • 注解配置。 项目中 */*Dao 、 */*Service 、 */*Api 都是这种配置。
